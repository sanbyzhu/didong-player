# 洞听播放器

洞听播放器是一个安卓优先的本地音频/视频播放器。当前版本使用 AndroidX Media3/ExoPlayer 作为播放核心，支持扫描本地文件夹、按数字/文件名排序、文件夹播放、自建列表、最高 8 倍速、播放位置记忆、文件夹级音频/视频独立倍速记忆、单曲循环、列表循环、AB 循环记忆、音量增益、TXT 文本朗读和朗读背景音乐。

## 当前已实现

- 扫描 SAF 文件夹，递归识别常见音频/视频格式。
- 文件夹扫描在后台线程执行，大目录扫描时界面不会卡住，并防止重复扫描。
- 扫描时显示已遍历文件夹和已找到音视频数量，并补充识别 m4v、ts、m2ts、flv、wmv、amr、wma、aiff 等格式。
- 支持直接打开单个音频/视频文件，并维护“打开的文件”列表。
- 支持从文件管理器通过“打开方式”唤起洞听播放器播放音频/视频。
- 按文件名自然排序，文件名包含数字时优先按数字排序。
- 播放音频和视频，视频在内置画面区域播放；视频可点击画面进入全屏，通过底部“退出”或返回键缩回主界面。
- 音频/朗读默认显示橘黄色树洞视觉舞台，整体界面统一为深木色和暖橘色；视频播放时自动切回视频画面。
- 声波粒子收束在视觉舞台底部的一条声波线上，会跟随播放状态和音效增益/低音/立体感强度变化，不遮住朗读文字。
- 暂无歌词时显示默认封面信息；TXT 朗读时会在视觉舞台显示当前朗读文字，并像歌词一样滚动。
- 支持手动导入 LRC 歌词或 SRT 字幕，音频歌词会显示在树洞视觉舞台，视频字幕显示在视频控制台下方。
- 扫描文件夹时会自动匹配同名 `.lrc` / `.srt` 文件；支持字幕/歌词提前、延后和重置偏移。
- 扫描文件夹时会记录同文件夹歌词/字幕候选，当前文件可从“字幕候选”里快速切换或手动导入。
- 视频播放时自动保持屏幕常亮，全屏时使用洞听自定义控制台，不再让系统快进条遮挡画面。
- 主界面和播放页都有明确的视频全屏入口，默认按原比例适应屏幕，适合手机竖屏视频；设置中心可选择填充画面。
- 视频下方常驻洞听控制台，全屏时点画面可唤出底部控制栏，支持播放/暂停、快退/快进、倍速、循环、AB 点、清 AB、适应/填充和退出全屏，播放中可自动隐藏。
- 视频支持双击播放/暂停、左右滑动快退/快进、右侧上下滑动调系统媒体音量，左侧上下滑动直接调当前窗口亮度。
- 视频全屏控制栏显示倍速和播放模式，并可快速切换适应/填充画面。
- 视频控制栏新增画面旋转，适合临时修正横竖屏方向不对的视频。
- 视频控制栏新增画中画入口；每个视频会单独记忆画面旋转和适应/填充模式，小窗会保持纯视频画面，退出后恢复正常视频窗口。
- 顶部播放进度条，可拖动跳转播放位置。
- 明显的播放/暂停按钮、上一首、下一首。
- 点击当前正在播放的列表项或标题可暂停/继续。
- 新增“播放页”入口，集中控制进度、上一首/下一首、快退/快进、倍速、循环、AB 点、书签、收藏和随机播放，并显示音频默认封面、AB 状态和书签数量。
- 播放页已拆成独立页面，通知栏或主界面返回时不会重新初始化正在播放的队列；页面内可直接调速、快进快退和切换循环模式。
- 独立播放页中间区域会同步显示当前歌词/字幕行，或 TXT/JSON/EPUB 朗读段落。
- 快退 15 秒、快进 30 秒。
- 搜索当前列表。
- 当前列表支持“全部媒体 / 仅音频 / 仅视频”快速筛选，并显示音频/视频数量和当前显示数量。
- 媒体列表改为点击“媒体列表”弹窗显示，当前播放项会高亮，避免列表常驻挤占播放控制区。
- 高级控制可折叠/展开，减少主界面拥挤，并记忆展开状态。
- 随机播放。
- 播放模式统一为顺序播放、单曲循环、列表循环、随机循环，并接入 ExoPlayer 的 repeat/shuffle 行为。
- 睡眠定时关闭，支持 15/30/60 分钟、播完当前停止和播完列表停止。
- 启动后恢复上次列表、上次文件、上次播放位置和倍速。
- 每个播放列表会单独记忆上次播放的文件和位置，切到别的列表再回来可继续上次位置。
- MediaSession 基础集成，为锁屏、耳机键和后续通知栏控制打底。
- 前台播放服务：播放时显示系统通知，支持通知栏播放/暂停、上一首、下一首、停止。
- 播放器核心迁入后台服务，离开界面后可继续播放。
- 播放失败会提示错误，并在列表中自动尝试下一首。
- 支持耳机拔出自动暂停、系统音频焦点处理和洞听专属通知小图标。
- 系统音效增强：响度增益、低音增强、立体感增强，并显示当前音效是否成功启用。
- 新增人声、低音、清晰、夜间、影院、学习等音效预设，预设会同步调整增益、低音、立体感和五段 EQ。
- 新增五段均衡器，从低频到高频可微调；仍通过系统音效接口实现，部分输出设备可能不支持。
- 0.25x 到 8.0x 倍速，按文件夹和媒体类型分别记忆默认倍速。
- 播放位置记忆。
- 单曲循环、列表循环，循环模式会记忆。
- AB 循环，按媒体文件记忆。
- AB 循环支持 A/B 点各自按 1 秒微调。
- 当前媒体支持时间点书签：添加书签、查看书签、跳转书签、删除书签。
- 支持一键回到开头、清除当前媒体播放位置。
- 自建播放列表，把当前扫描结果加入列表。
- 扫描文件夹后自动生成“全部文件”和每个子文件夹对应的播放列表，并持久保存。
- 内置“收藏”和“最近播放”列表，支持收藏当前、移出列表、清搜索、清空最近播放。
- 当前文件或长按列表项可加入到指定普通播放列表。
- 列表管理支持刷新上次文件夹、当前列表排序/倒序、重命名/删除普通列表，自动扫描列表和内置列表受保护。
- 列表管理新增最近扫描文件夹和增量刷新入口，可快速回到之前授权过的目录。
- 列表管理支持清理当前列表中已移动、删除或授权失效的文件。
- 列表管理支持全库清理失效文件、批量收藏当前列表、移除重复文件、只保留音频或只保留视频。
- 列表项支持长按菜单：播放/暂停、收藏、移出当前列表、上移、下移。
- 播放音量和 LoudnessEnhancer 增益，增强声音但不修改源文件。
- 导入 txt 文本，使用系统 TextToSpeech 语音朗读。
- 长 txt 自动分段朗读，显示朗读进度，读完自动停止背景音乐。
- 朗读语速、音调、人声选择和朗读进度可记忆，设置里可从头朗读。
- 背景音乐支持多首列表、播放/暂停、下一首、停止、删除和清空；既可用于 TXT 朗读，也可在听普通音频时作为轻背景音伴奏。
- 背景音乐音量可在主界面高级控制和设置中心实时调整，后台朗读服务播放中的背景音也会同步生效。
- TXT 朗读支持后台服务，主界面“朗读/暂停”可开始、暂停、继续，锁屏/切后台后也可通过通知栏上一段、暂停/继续、下一段、从头读、停止；主界面可查看分段并从指定段落开始朗读。
- TXT 朗读支持文本搜索定位和段落书签，便于长文本继续学习。
- 文本导入支持 TXT、JSON 和轻量 EPUB 解析；JSON 会整理字段文本，EPUB 会提取 html/xhtml/txt 章节正文。
- 朗读书签支持添加、列表查看、跳转、删除和去重保存。
- TXT 朗读新增章节目录识别，可按“第 X 章/节/回/卷”或英文 Chapter 跳转，并显示朗读进度统计。
- TXT 新增书架，导入过的文本会记录到书架中，可从上次朗读段落继续打开。
- 人声选择支持“男声优先”尝试匹配系统 TTS 男声，并提供系统 TTS 设置入口安装或切换更多语音包。
- 设置面板支持打开系统声音设置、恢复音效默认、恢复朗读默认、清除当前播放位置和关闭睡眠定时。
- 新增设置中心，集中管理播放、音效滑杆、朗读滑杆、视频开关、恢复全部默认和数据清理入口。
- 新增设备体验自检，集中查看通知权限、后台播放服务、MediaSession、音效支持、画中画和 TTS 状态。
- 新增数据备份/恢复，可导出和导入播放列表、播放记忆、字幕绑定、书签、TXT 书架和数据库镜像。
- 播放列表保存时会保留上一份 JSON 备份，主记录异常时可自动尝试恢复。
- 新增 SQLite 本地数据库镜像，播放列表、播放位置、列表记忆、字幕、媒体书签和朗读书签会多写一份兜底记录，提高异常恢复能力。
- 通知栏增加播放进度显示，锁屏和后台时更容易确认当前进度。

## 构建

## V0.0.12 更新
- TXT 朗读和普通媒体播放进一步分离，独立播放页在朗读状态下会显示朗读标题、朗读进度和朗读控制。
- TXT 朗读支持进度条按段跳转，并在独立播放页新增目录、书签、添加书签、搜索、上一段和下一段入口。
- 首页树洞视觉区改为跟随当前朗读位置显示，不再按时间轮播导致文字对不上。
- 朗读文本支持系统 TTS `onRangeStart` 字词级回调检测，设备自检里会显示高亮能力状态。
- 背景音乐管理面板升级为可视列表，支持选中播放、删除当前、上移、下移、设默认、清空、单曲循环和列表循环，并在首页显示当前背景音乐状态。

## Windows 尝鲜版
- 已新增免安装网页原型：[DongtingPlayer.html](windows/DongtingPlayer.html)。
- Windows 上可双击 [启动洞听Windows版.bat](windows/启动洞听Windows版.bat)，或直接用 Edge/Chrome 打开 `windows/DongtingPlayer.html`。
- 当前支持扫描文件夹、按文件夹生成列表、自然数字排序、音频/视频播放、0.25x 到 8x 倍速、按文件夹和媒体类型记忆倍速、播放位置记忆、单曲循环、列表循环、AB 循环、TXT/JSON 朗读和背景音乐。
- 这是第一版桌面尝试，基于浏览器本地能力实现；浏览器出于安全限制不会永久保存文件授权，下次打开通常需要重新扫描文件夹。

## 可选 MultiTTS 接口
- 设置中心新增“使用MultiTTS接口”，会把 TXT 朗读分段发送到 `http://127.0.0.1:8774/forward` 并播放返回音频。
- MultiTTS 未启动、接口失败或返回不可播放音频时，会自动退回现有系统 TTS 朗读，不影响原功能。
- 需要 MultiTTS 在手机后台运行，并建议在系统里给 MultiTTS 关闭电池优化。

```powershell
.\gradlew.bat assembleDebug
```

生成的 APK 位于：

```text
app\build\outputs\apk\debug\app-debug.apk
```

## 参考方向

实现参考了 GitHub 上常见 Android 音乐/视频播放器的功能组织方式，并采用 Android 官方推荐的 Media3/ExoPlayer 播放栈。没有直接复制第三方项目代码，避免许可证、架构耦合和后续维护问题。

## 后续路线

1. 拆分架构：已先抽出媒体识别、数字排序、时间格式和部分播放列表存储工具，后续继续把播放控制、TTS、列表管理拆成独立类。
2. 数据库：从 SharedPreferences/JSON 迁移到 Room，支持更大的媒体库和更稳定的查询。
3. 高质量人声：接入可授权的离线语音包或云端 TTS，提供更自然的普通话/英文人声。
4. Windows：优先评估 Kotlin Multiplatform + Compose Desktop 或 Flutter，复用媒体库数据结构。
5. 投屏：根据设备能力接入 MediaRouter、Google Cast、DLNA 或 Miracast 路径；功能目标包括“只投画面、声音留手机”和“音画都投影”，控制仍保留在手机端。
